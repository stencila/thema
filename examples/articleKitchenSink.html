Abstract

This is our "kitchen sink" example which we use when developing, demoing, and testing themes. It is intended to include examples of most of the types of elements in a Stencila document (whilst staying relatively brief). This abstract can include not just string content, but other content types as well, for example, emphasized text, an inline math fragment: E=mc2. Inline content

Primitive types

The JSON primitives are valid inline content. This includes: null and true and false and numbers like 42 and 3.14, as well as plain old strings like this one.

Mark types

The are several types of mark: emphasis and strong emphasis and deleted content and quoted content and subscripted content and superscripted content.

Citation types

Citations have a 'mode' which defaults to 'Parenthetical', for example, 1Alvarez and Fyffe2007 and 2Asahina et al.2014 and 3Baines et al.2001; can be 'Narrative' 1Alvarez and Fyffe2007; can be 'NarrativeAuthor' (only show author/s) 1Alvarez and Fyffe2007; can be 'NarrativeYear' (only show year) 1Alvarez and Fyffe2007.

Citations can also have a prefix e.g.1Alvarez and Fyffe2007, a suffix 1Alvarez and Fyffe2007p. 21, or both e.g.1Alvarez and Fyffe2007p. 42. They can also have 'content' (which overrides prefix, suffix, author, year) as published by us in twenty-oh-seven.

Citations can be grouped like 1Alvarez and Fyffe20072Asahina et al.2014 and 1Alvarez and Fyffe20072Asahina et al.20143Baines et al.2001 in which case themes will usually place parentheses or brackets around the group, and commas or semicolons between them.

Code types

There are two inline code types: code fragments (static) and code expressions (executable); and two block code types: code blocks (static) and code chunks (executable).

Code fragments

A code fragment can have no language defined, e.g. some code. Or, it can have a language, in which case it should be syntax highlighted: func(arg="foo")

Code expressions

A code expression, may or may not have a language (at time of writing at least). It may have output e.g. 2 * 24; it may have errors but no output e.g. fooSometimes the expression and output may be really longa_function_with_a_really_long_name(what = a_variable_that_also_has_a_really_long_name)wow, this is a long string that a_function_with_a_really_long_name outputted when called with a_variable_that_also_has_a_really_long_name .

Code blocks

A code block will usually have a language specified e.g.

// Some Javascript
function beep() {
  console.log('boop')
}

But it may not e.g.

# Some Python
def beep():
  print('boop')

Code chunks

A code chunk can have varying number and types of outputs e.g.

// No output
const x = 42
 
# String output
'Hello world'
 
# Datatable output
head(mtcars)
 
# ImageObject output
 plot(sin(seq(0,4, 0.01)))
 
# A number, a string, an array
42
'Hello'
[1, 2, 3]
